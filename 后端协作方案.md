## 三人协作执行方案（融合版：保留分工表 + 仅 Gate 流程）

### Summary
- 覆盖 3 个核心问题并统一到一个协作闭环：`SOP 重复触发`、`UserState 并发写丢失`、`BM25 冷启动竞态`。
- 执行节奏：`阶段串行`，`阶段内并行`，跨阶段必须过 Gate 才能进入下一阶段。
- 协作流程只保留 Gate 规则，不再额外引入日常流程条款。

### 任务分工总览（3 问题全覆盖）
| 问题 | 主负责 | 配合 | 主要阶段 | 完成判定 |
|---|---|---|---|---|
| SOP 重复触发 | Dev A | Dev B + Dev C | Phase0.5 / Phase3 | 重复触发率可观测，且灰度期保持低风险 |
| UserState 并发写丢失 | Dev A | Dev B + Dev C | Phase1 | 同用户并发写入 0 丢失 |
| BM25 冷启动竞态 | Dev B | Dev A + Dev C | Phase1.5 | 多 worker 冷启动加载受控且稳定 |

### 每阶段分工（实操）
| 阶段 | Dev A（状态/规则） | Dev B（运行时/平台） | Dev C（测试/验收） | 交付物 |
|---|---|---|---|---|
| Phase0.5（1天） | 梳理 SOP 触发状态机与失败路径 | 补齐 trace_id/goal_id/user_id 日志字段 | 建 SOP 场景矩阵并自动化 | SOP 风险清单 + 场景矩阵 + 日志字典 |
| Phase1（3-5天） | 实现 UserState 并发控制与冲突处理 | 对齐调用链重试与异常语义 | 并发回归用例（先红后绿） | 并发实现 + 回归报告 |
| Phase1.5（1-2天） | 评审 BM25 修复一致性影响 | 实现冷启动并发保护与降级 | 冷启动压测与多 worker 验证 | 竞态修复 + 压测报告 |
| Phase3（3-7天灰度） | 收敛 SOP/State 线上风险 | 告警与仪表盘闭环 | 灰度日报与复盘证据 | 7天灰度报告 + 问题闭环记录 |

### Gate 规则（细化版，唯一协作流程）
| Gate | 时点 | 前置条件 | 必交产物 | 验证方式 | 通过标准 | 不通过处理 | 签字人 |
|---|---|---|---|---|---|---|---|
| Gate0：SOP 基线冻结 | Phase0.5 结束 | Phase0 已在 main，且可复现 SOP 流程 | 状态机图、异常路径表、日志字段清单、SOP 场景矩阵 | 场景逐条回放 + 日志对照 | 场景可复现、异常可定位、字段完整 | 回到 Phase0.5 补齐，不得进 Phase1 | Dev A + Dev C |
| Gate1：并发合同冻结 | Phase1 Day1 结束 | Gate0 通过 | 并发合同文档（异常名、重试次数、冲突语义、日志键） | 合同评审会 + 样例请求回放 | 三人对合同无歧义、实现边界明确 | 暂停编码，先统一合同 | Dev A + Dev B |
| Gate2：UserState 并发验收 | Phase1 收尾 | Gate1 通过，相关 PR 已关联测试 | 并发测试报告、关键字段不覆盖证明、失败样例清零记录 | `pytest -q backend/tests/integration/test_concurrent_state.py` 连跑 3 轮 | 同用户并发写 0 丢失，关键回归全绿 | 回退到 Gate1 合同修正，再重测 | Dev A + Dev C |
| Gate3：BM25 冷启动验收 | Phase1.5 收尾 | Gate2 通过 | 冷启动压测报告、加载次数日志、降级路径日志 | `pytest -q backend/tests/integration/test_bm25_cold_start.py` + 压测脚本 | 每 worker 冷启动加载受控（一次级别），首波延迟稳定 | 暂停合并，先修竞态/降级路径 | Dev B + Dev C |
| Gate4：三问题灰度闭环 | Phase3 收尾 | Gate3 通过并进入灰度 | 7天灰度数据、告警记录、复盘结论 | 指标复核 + 故障注入复验 | 无 P0，三问题均有“发现-修复-验证”证据链 | 继续灰度，不允许全量 | Dev A + Dev B + Dev C |

### 接口与实现边界（避免范围漂移）
- 外部 API 默认不变；本轮以内部一致性和稳定性为主。
- 允许新增测试与观测字段；不做大规模存储迁移。
- Phase2（迁移类工作）不在本轮强制执行范围内，仅在 Gate4 后按触发条件单独立项。

### 验收测试清单（按 3 核心问题）
1. SOP：同一 goal 重复触发，第 2 次跳过初始化；并发触发下无重复初始化。  
2. UserState：同用户并发写（10 worker）0 丢失；`tags/milestones/quoted/session_id` 无覆盖回退。  
3. BM25：冷启动并发查询下加载受控；故障路径可降级且可观测。  

### Assumptions（默认）
- 当前 `main` 已包含 Phase0 幂等补丁，作为本计划基线。
- 3 人可同步参与评审与签字；未签字视为 Gate 未通过。
- 如需引入 DB/Redis 等架构升级，另开新变更单，不并入本计划。
